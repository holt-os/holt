# Contributing to Holt

Thanks for your interest! Holt is designed so you can extend it **without touching the core**.

> Holt is in early development (Phase 0). The core is still taking shape, so the smoothest way to help right now is with ideas, issues, and plugin/skill design feedback.

## Ways to contribute

- **Skills**: Holt uses the [agentskills.io](https://agentskills.io) format. A skill is largely Markdown; you don't need to read the core to write one.
- **Plugins**: providers, channels, embeddings, and tools are [MCP](https://modelcontextprotocol.io) servers. Write them in any language that speaks MCP.
- **Core**: TypeScript. Small and strongly typed. See [`ARCHITECTURE.md`](./ARCHITECTURE.md).

## Getting started (dev)

```bash
git clone https://github.com/holt-os/holt.git
cd holt
npm install
npm run dev            # runs the CLI from source
npm run typecheck
```

## Development

Holt writes to `~/.holt` (trust list) and `<cwd>/.holt` (per-folder config and memory), so it plays well with git worktrees: check out a branch in a separate worktree, `npm install` there, and work in isolation without disturbing your main checkout.

Run the CLI from source without building:

```bash
npm install
npm run dev init
npm run dev chat
```

**Test with an isolated HOME and a throwaway workspace.** Point `HOME` at a temp dir so you never touch your real trust list, and pre-seed a trusted workspace so `holt` skips the trust prompt:

```bash
export HOME=/tmp/holt-test
mkdir -p "$HOME/.holt"
ws=/tmp/holt-ws && mkdir -p "$ws"
# pre-trust the workspace so ensureTrusted() passes non-interactively
printf '{"trusted":["%s"]}\n' "$ws" > "$HOME/.holt/trust.json"
cd "$ws"
```

**The echo-brain trick.** You do not need a real model to exercise the REPL. Set a brain's `command` to `echo` in the workspace `config.json`; `runBrain` will spawn `echo "<prompt>"` and stream the rendered prompt straight back as the "reply". That lets you test trust, recall, transcript replay, brain switching, and memory writes with no model installed:

```json
{
  "version": 2,
  "defaultBrain": "claude",
  "brains": {
    "claude": { "id": "claude", "label": "Echo", "command": "echo", "args": [], "enabled": true },
    "codex":  { "id": "codex",  "label": "Codex (OpenAI)", "command": "codex",  "args": ["exec"], "enabled": false },
    "gemini": { "id": "gemini", "label": "Gemini CLI",     "command": "gemini", "args": ["-p"],   "enabled": false }
  }
}
```

Write that to `<ws>/.holt/config.json`, then pipe input into `npm run dev chat` (or `node dist/cli.js chat`) to see the prompt Holt built.

## Ground rules

- Keep the core small. If it can be a plugin, make it a plugin.
- Never commit secrets or personal data. Per-folder `.holt/` (config and memory) is where your data lives; keep it out of commits.
- Open an issue to discuss larger changes before a big PR.
- **No em-dash characters in code or docs.** CI greps for them and fails the build if it finds one. Use commas, colons, or parentheses.
- **Plain commit messages.** No tool-attribution footers or "generated by" trailers.
