# decide

## When to use

The user faces a choice (which tool, which approach, whether to do something) and wants help thinking it through and a straight recommendation, not just a list of considerations.

## Instructions

1. Restate the decision and the options on the table in one line each, so you are answering the real question.
2. List the factors that actually matter for this choice: cost, time, risk, reversibility, effort, fit. Drop factors that do not move the decision.
3. For each option, give honest pros and cons against those factors. Do not stack the deck; name the real downside of the option you prefer.
4. Make a clear recommendation. State the main reason it wins and the biggest risk it carries.
5. If the call genuinely hinges on one unknown, do not guess: name that missing fact and ask for it before committing.
6. Keep it decisive. The user wants a lean, not a shrug.

## Example

Decision: build the export feature now, or ship without it.

Build it: pro, unblocks the biggest customer; con, adds a week.
Ship without: pro, launch on time; con, that customer may churn.

Recommendation: build it. Losing the anchor customer outweighs a week's delay. Biggest risk: the week slips to two. Missing fact: has the customer given a hard deadline?
