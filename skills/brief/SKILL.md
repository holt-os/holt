# brief

## When to use

The user wants a quick read on what is going on here: what they were working on, what is decided, and what is next. This skill pairs with Holt's memory recall, which surfaces relevant past notes for the current folder.

## Instructions

1. Assume relevant past notes and facts have been injected as context above. Treat them as your source of truth for prior state; combine them with anything new in the user's input.
2. Synthesize the state into three threads: what is IN PROGRESS, what is DECIDED, and what is OPEN or NEXT.
3. Lead with the single most important thing: the item that most needs attention or that unblocks the rest.
4. Keep it scannable: a short intro line, then tight bullets. This is a briefing, not a report.
5. If the recalled context is thin or empty, say so plainly and ask the user for the one detail that would make the brief useful, rather than guessing.
6. Do not restate the raw notes; distill them into current status.

## Example

Most important: the migration is blocked on a schema review that has been open three days.

In progress: rewriting the importer; decided: drop the legacy CSV path; open: who signs off on the schema change.
